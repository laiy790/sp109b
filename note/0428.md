## Process & Thread
```
行程
→傳統的兩個行程通常擁有不同的記憶空間,在具有記憶體管理單元(MemoryManagement Unit:MU)的作業系統中,兩個行程的記憶空間是完全獨立的,各自擁有自己的記憶體映射表。所以在行程切換的同時也必須更換映射表,這樣的動作會消耗許多時間。

執行緒
→執行緒之間共用記憶體空間
→切換時不需更換映射表
→切換動作非常快速。
```

## malloc動態記憶體配置
```
malloc 代表 memory allocation，用來配置指定大小的記憶體空間，傳回新空間第一個位元組的記憶體位址，配置的空間處於尚未初始化的狀態。
```
